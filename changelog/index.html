Build Log

Changelog

This isn't a changelog for a SaaS product. It's a build log.

RiskCast exists because we're building the thing we're talking about. OpenClaw is a self-hosted, multi-agent AI system that runs real business operations — email triage, event inquiry responses, SEO content, meeting digests — across two Mac Minis at Tabbris Innovation Center in Charlotte, NC.

This page documents what actually shipped. Not roadmaps, not intentions — what got built, what broke, and what we fixed. Every entry traces back to a real session, a real commit, or a real failure.

2026-03-20
Changed
  • Upgraded to OpenClaw v2026.3.13 (from v2026.3.8) — 5 patch releases of accumulated fixes including session pruning, model failover improvements, and plugin lifecycle changes
  • Primary model restored to Anthropic Claude Sonnet 4.6 for Alfred and Drafting agents — Gemini 2.5 Flash demoted to first fallback, Qwen local as second. Cross-provider fallback chain now: Sonnet → Gemini Flash → Qwen (three providers, no single-vendor outage takes down the system)
  • Enabled sandbox mode non-main for Monitor and Drafting agents — OS-level process isolation in addition to the existing tool deny-lists. Main agent (Alfred) remains unsandboxed for full operational access
  • Removed per-agent sandbox: off overrides that were silently defeating the global sandbox policy
Added
  • Lobster workflow engine enabled — OpenClaw's native deterministic pipeline runtime, replacing the need for multi-script shell orchestration. Typed JSON pipelines with approval gates and resume tokens
  • llm-task plugin enabled — allows Lobster workflows to embed LLM classification/drafting steps inside otherwise deterministic pipelines (e.g., Qwen classifies email within a Lobster workflow without the LLM deciding what step comes next)
  • First Lobster workflow written: email-triage.lobster — 5-step pipeline: fetch from Gmail → classify via Qwen local → format/route → notify #ta_events-control → notify #ta_mission-control. Approval gates on both Slack notification steps. Can be triggered on-demand via conversation, on cron, or via system event — not cron-only like the shell scripts
  • Created ~/.openclaw/workspace/workflows/ directory for Lobster workflow files
  • RiskCast podcast production pipeline — built riskcast-produce skill for Alfred with four-stage workflow: intake (raw audio drop) → transcription (whisper.cpp on Apple Silicon, daily 1 AM cron + on-demand) → paper edit (Alfred generates segment-by-segment edit template from transcript) → editor handoff (assembled package with audio, transcript, paper edit, show notes draft, and editor brief). Full directory structure deployed at ~/.openclaw/riskcast/. Transcription script at riskcast-transcribe.sh. Alfred is now an integral part of the production loop — on brand for a podcast about AI agents
  • Ran openclaw doctor and openclaw security audit --deep — baselined system health post-upgrade. 0 critical findings
Known Issues
  • Security audit flags Slack group allowlist as potential multi-user heuristic — expected and acceptable for the personal-assistant trust model
2026-03-18
Fixed
  • Drafting agent pipeline restored — a March 11 Claude session had created surface-to-slack.sh as a "pipeline unblock" that bypassed the LLM drafting agent entirely, posting raw notifications to Slack. The script was added to cron but never removed or documented, silently starving the drafter of work
  • Root cause: surface-to-slack.sh grabbed pending items before drafting-queue.sh could process them. All three OpenClaw cron jobs in jobs.json were also disabled
  • Restored drafting-queue.sh to system cron (1 min after classifier), shifted surface-to-slack.sh to 5 min later as a fallback
  • Rewrote surface-to-slack.sh to prefer drafted items (includes draft text in Slack posts), with a 10-min stale fallback for items drafting missed
